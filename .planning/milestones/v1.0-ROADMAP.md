# Milestone v1.0: Semantic Search

**Status:** ✅ SHIPPED 2026-03-21
**Phases:** 1-3
**Total Plans:** 9

## Overview

`tree-sitter-mcp` v1.0 shipped a standalone, AI-agent-first MCP server for semantic code search over local workspaces. The milestone established local stdio transport, deterministic workspace discovery, explicit capabilities/health diagnostics, definition-centric retrieval, and finished with reference search, context snippets, pagination, and agent-ready response shaping.

## Phases

### Phase 1: Server Foundation and Workspace Discovery

**Goal**: Deliver a runnable `tree-sitter-mcp` package with stdio-based MCP connectivity, workspace targeting, grammar awareness, exclusion handling, and actionable diagnostics.
**Depends on**: Nothing (first phase)
**Plans**: 3 plans

Plans:

- [x] 01-01: Scaffold the TypeScript MCP server package, runtime configuration, and SDK adapter boundary in `tree-sitter-mcp`.
- [x] 01-02: Implement workspace discovery, ignore/exclusion rules, and grammar/language registration.
- [x] 01-03: Add capability/health tooling plus parse and unsupported-language diagnostics.

**Details:**
Built the standalone package and stdio bootstrap, then added normalized workspace root handling, exclusion policy, builtin grammar registration, and user-facing tools for workspace setup, capabilities, health, and file/workspace symbol search with structured diagnostics.

### Phase 2: Definition Search Core

**Goal**: Turn Tree-sitter parses into reliable definition-oriented semantic search tools with precise locations and useful filters.
**Depends on**: Phase 1
**Plans**: 3 plans

Plans:

- [x] 02-01: Build the on-demand parse and query pipeline for definition extraction.
- [x] 02-02: Normalize symbol metadata, source ranges, and filter handling across languages.
- [x] 02-03: Expose definition-search MCP tools with read-only semantics and structured payloads.

**Details:**
Layered definition extraction and resolution on top of the Phase 1 parser/workspace snapshot, normalized paths/ranges/filter semantics across languages, and exposed `search_definitions` plus `resolve_definition` as read-only MCP tools with end-to-end stdio coverage.

### Phase 3: Reference Search and Agent-Ready Results

**Goal**: Complete the v1 semantic-search experience with reference lookup, surrounding syntax context, pagination, and MCP responses tuned for AI agents.
**Depends on**: Phase 2
**Plans**: 3 plans

Plans:

- [x] 03-01: Implement reference and call-site retrieval on top of the definition pipeline.
- [x] 03-02: Add context extraction, snippet shaping, and pagination metadata.
- [x] 03-03: Validate end-to-end agent workflows and package the server for standalone use.

**Details:**
Implemented definition-anchored reference search, reusable context/snippet helpers, explicit offset-based pagination, and the read-only `search_references` MCP tool. The phase also finished standalone CLI packaging and end-to-end agent workflow coverage over stdio.

## Milestone Summary

**Decimal Phases:**

- None.

**Key Decisions:**

- Use a standalone `tree-sitter-mcp` package so MCP clients can launch the server without repo-root glue.
- Keep v1 stdio-first and local-only rather than introducing additional transports before the core workflow proved useful.
- Prefer on-demand parsing and workspace snapshots over persistent indexing for the first shipped milestone.
- Use deterministic exclusions, explicit capabilities/health tools, and shared structured diagnostics so agent workflows stay debuggable.
- Keep definition and reference tools as thin MCP adapters over shared backend services to preserve one source of truth.

**Issues Resolved:**

- Closed the MCP transport explicitly in bootstrap tests to prevent hanging subprocesses.
- Pinned a compatible Tree-sitter parser/grammar dependency set after peer dependency conflicts.
- Corrected reference-tool pagination assertions to match the stable pagination contract from Phase 03-02.

**Issues Deferred:**

- Persistent index support for faster repeated queries.
- Streamable HTTP transport and broader integrations beyond local stdio.
- Multi-repository search, relationship views, and write/refactor workflows.

**Technical Debt Incurred:**

- Builtin grammar support is intentionally limited to JavaScript, TypeScript/TSX, and Python in v1.
- On-demand re-parsing may need optimization or indexing for large repositories.
- Higher-level relationship views (call graphs, dependency chains, impact analysis) remain future work.
